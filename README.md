# ChoreSync

A personal household chore app for two people, built with **SwiftUI + Core Data + CloudKit**.  
100% free — no backend, no subscription. Data syncs through iCloud using **CloudKit Sharing**, so each person keeps their own Apple ID.

---

## Features

| Feature | Details |
|---|---|
| **Categories** | Group chores into colour-coded, icon-tagged buckets (Kitchen, Bathroom, Outdoor, etc.) |
| **Assignments** | Assign each chore to You, Your Partner, or Both |
| **Due dates** | Pick a specific date, any week, any month, or no due date — supports future weeks and months |
| **Repeating chores** | Daily, weekly, bi-weekly, monthly, or yearly — chore resets automatically after completion |
| **Completion tracking** | Per-person history stored for every completion |
| **CloudKit Sharing** | Each person uses their own Apple ID. One person invites the other — no shared account needed |
| **Shopping list** | Separate tab for groceries/household items — group by store or by type, with assignee support |
| **Easy UI** | Swipe to complete or delete; tap to edit; colour-coded urgency sections |

## Getting Started

### 1. Install Xcode

Download **Xcode 16** from the Mac App Store or from [developer.apple.com/download](https://developer.apple.com/download).

### 2. Open the project

```bash
open ~/Coding/ChoreSync/ChoreSync.xcodeproj
```

Or double-click `ChoreSync.xcodeproj` in Finder.

### 3. Run on the Simulator (no developer account needed)

1. Select an iPhone 16 simulator from the device picker in Xcode's toolbar.
2. Press **Cmd+R** (or the play button).
3. The app launches with six default categories pre-seeded.

### 4. Run on a real device (requires developer account)

1. Connect your iPhone via USB.
2. In Xcode: **Signing & Capabilities** tab → select your Team.
3. Xcode will auto-provision a development certificate.
4. Select your device from the picker and press **Cmd+R**.

## Setting Up CloudKit Sharing (Two Separate Apple IDs)

This is the recommended setup. Each person keeps their own Apple ID. One person creates the household and invites the other.

### Step 1: Configure CloudKit (one-time, in Xcode)

1. **Create the iCloud container**:
   - Go to [developer.apple.com](https://developer.apple.com) → Certificates, Identifiers & Profiles → iCloud Containers.
   - Click **+** and create a container: `iCloud.com.choresync.app`

2. **Link it to your App ID**:
   - Under Identifiers, find your App ID.
   - Enable **iCloud** and select the container.

3. **Configure in Xcode**:
   - Open ChoreSync.xcodeproj → select the **ChoreSync** target.
   - **Signing & Capabilities** tab → **+ Capability** → **iCloud**.
   - Check **CloudKit** and select your container.

4. **Initialise the CloudKit schema** (first run only):
   - In `PersistenceController.swift`, find this block:
     ```swift
     // #if DEBUG
     // try container.initializeCloudKitSchema(options: [])
     // #endif
     ```
   - Uncomment it, build and run once, then comment it back out.

5. **Deploy to production**:
   - In CloudKit Console → select your container → **Deploy to Production**.

### Step 2: Install on both phones

Build and install ChoreSync on both iPhones. Each person should be signed into their own Apple ID.

### Step 3: Person A invites Person B

1. **Person A** opens ChoreSync → **Settings** tab → **Household Sharing** section.
2. Taps **"Invite Partner"**.
3. iOS shows the standard sharing sheet — send via iMessage, email, AirDrop, or copy the link.
4. **Person B** receives the link and taps it.
5. iOS prompts Person B to accept the share → they tap **Accept**.
6. Both phones now show the same chores. Changes sync both ways automatically.

### How it works under the hood

- ChoreSync uses two Core Data persistent stores:
  - **Private store** — syncs to the owner's private CloudKit database
  - **Shared store** — syncs to CloudKit's shared database
- When Person A creates a share, all data in their private zone becomes accessible to Person B via the shared database.
- `@FetchRequest` automatically queries both stores, so chores from both people appear in the same list.
- `NSPersistentCloudKitContainer` handles conflict resolution — the most recent edit wins.

## Alternative: Shared iCloud Account

If you and your partner share a single Apple ID (common for family purchases), CloudKit sharing is not needed. Both devices automatically sync via the private store. No invite step required — just install the app on both phones and everything syncs.

## Data Model

### Chore

| Field | Type | Description |
|---|---|---|
| `id` | UUID | Primary key |
| `title` | String | Short chore description |
| `notes` | String? | Optional detail |
| `assignedTo` | Int16 | 0=Me, 1=Partner, 2=Both |
| `dueDateType` | Int16 | 0=Specific date, 1=Week, 2=Month, 3=None |
| `dueDate` | Date? | Set for all non-None types; stores a date within the selected week/month |
| `repeatInterval` | Int16 | 0=None, 1=Daily, 2=Weekly, 3=Biweekly, 4=Monthly, 5=Yearly |
| `isCompleted` | Bool | Whether the chore is done |
| `completedAt` | Date? | When it was completed |
| `completedByMe` | Bool | Completion flag for person 1 |
| `completedByPartner` | Bool | Completion flag for person 2 |

### Category

| Field | Type | Description |
|---|---|---|
| `id` | UUID | Primary key |
| `name` | String | Display name |
| `colorHex` | String | Hex colour string e.g. `#FF6B6B` |
| `iconName` | String | SF Symbol name e.g. `fork.knife` |

### CompletionLog

Each time a chore is marked complete, a `CompletionLog` record is inserted.  
This powers per-person history and future stats/streaks features.

### ShoppingItem

| Field | Type | Description |
|---|---|---|
| `id` | UUID | Primary key |
| `name` | String | Item name |
| `quantity` | String? | Optional quantity (e.g. "2", "1 gal") |
| `store` | String? | Store name (e.g. "Costco", "Target") |
| `itemType` | String? | Item category (e.g. "Food", "Household") |
| `assignedTo` | Int16 | 0=Me, 1=Partner, 2=Both |
| `isPurchased` | Bool | Whether the item has been bought |
| `purchasedAt` | Date? | When it was purchased |
| `notes` | String? | Optional notes |
| `sortOrder` | Int32 | Manual ordering |
| `createdAt` | Date | Creation timestamp |

## Repeat Logic

ChoreSync uses a **rolling repeat** model:

- When you mark a repeating chore complete, it is **not archived** — its due date advances by the interval from the completion date.
- The completion is recorded in `CompletionLog` for history.
- Example: a weekly "vacuum" chore completed on a Wednesday gets a new due date of the following Wednesday.

Non-repeating chores are archived in place (`isCompleted = true`) and shown in the **Completed** section.

## Customising Person Names

1. Open the app → **Settings** tab.
2. Tap the name fields and type your preferred names.
3. Names update everywhere in the app instantly (stored in UserDefaults).

Note: Person names are stored locally on each device (not synced via CloudKit). Each person can set their own "Me" and "Partner" labels independently.

## Known Limitations / Future Ideas

- **No push notifications** — due-date reminders could be added using `UNUserNotificationCenter`.
- **No widget** — a home screen widget showing today's chores via `WidgetKit`.
- **No drag-to-reorder** — the `sortOrder` attribute is in place; a `List` with `.onMove` would enable it.
- **No app icon** — add a 1024x1024 PNG to `Assets.xcassets/AppIcon.appiconset/` and update `Contents.json`.
- **No dashboard/scoreboard** — `CompletionLog` tracks per-person history; a stats view could show streaks and completion counts.
